Mise en place de la machine virtuelle

SAÉ S1.03 – BUT Informatique

IUT de Paris – Rives de Seine

Pour héberger notre machine virtuelle nous avons utilisé VirtualBox. VirtualBox a été choisi car il s’agit d’un outil de virtualisation gratuit, open source et multiplateforme. Il permet de créer et de gérer facilement des machines virtuelles avec une interface graphique intuitive.

VirtualBox offre également une bonne gestion des modes réseau (NAT, pont, réseau interne), indispensables pour la mise en place et le test d’un serveur web accessible depuis la machine hôte. Cela permet d’héberger et de consulter localement le site web du rapport.

Nous avons choisi Debian comme distribution Linux, car c’est celle utilisée dans l’IUT et nous avions déjà connaissance de son interface graphique.

Lors de la création de la machine virtuelle, nous avons dû choisir le type de connexion réseau. Le mode réseau en accès par pont (bridged) a été sélectionné afin que la machine virtuelle soit considérée comme un ordinateur à part entière sur le réseau local. Elle obtient donc sa propre adresse IP, ce qui est idéal pour l’hébergement d’un serveur web.

Personnalisation du terminal

Pour améliorer l’expérience utilisateur sur le terminal, nous avons personnalisé l’invite de commande grâce à la variable PS1 :

  • \e[45;37m : couleur de fond rose et texte blanc
  • \A : affiche l’heure au format HH:MM
  • \u : affiche le nom de l’utilisateur connecté
  • \w : affiche le répertoire courant
  • \$ : affiche $ pour un utilisateur normal ou # pour l’administrateur

Nous avons ajouté cette définition dans le fichier ~/.bashrc, puis rechargé la configuration avec source ~/.bashrc. Cela permet d’avoir le style défini à chaque ouverture du terminal :
