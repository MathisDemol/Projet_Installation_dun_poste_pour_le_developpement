Définition de la virtualisation

La virtualisation est une technique informatique qui permet d’utiliser un seul ordinateur ou serveur pour faire fonctionner plusieurs systèmes ou environnements en même temps. Chaque environnement est virtuel mais fonctionne comme s’il était réel, ce qui permet de mieux utiliser les ressources et de réduire les coûts.

Différence entre émulateur, simulateur et virtualisation

  • Émulateur : permet de faire fonctionner sur un appareil des programmes faits pour un autre appareil.
  • Simulateur : reproduit une situation pour s’entraîner ou faire des tests, sans que ce soit exactement la réalité.
  • Virtualisation : permet d’utiliser plusieurs systèmes différents sur un seul ordinateur, en même temps.

Type de virtualisation pour VirtualBox, VMware Workstation et Stratus

VirtualBox et VMware Workstation sont des hyperviseurs de type 2 (hébergés), installés comme logiciels sur un OS hôte (Windows, Linux ou macOS) et permettant d’exécuter plusieurs machines virtuelles en parallèle. Ils reposent sur les ressources du système hôte et sont principalement utilisés pour le développement, les tests et la formation.

Stratus utilise une virtualisation de type 1 (hyperviseur natif), qui s’exécute directement sur le matériel, sans système d’exploitation hôte. Ce type est conçu pour des environnements professionnels critiques, nécessitant haute disponibilité et performances optimisées.

Forme et lieu de sauvegarde de la machine virtuelle

La machine virtuelle est sauvegardée sous forme d’un ensemble de fichiers sur le système hôte. Ces fichiers contiennent le disque virtuel et les paramètres de configuration. Par défaut, ils sont enregistrés dans le répertoire personnel de l’utilisateur, dans un dossier dédié aux machines virtuelles.

Choix techniques pendant la création de la machine virtuelle

  1. Logiciel de virtualisation : VirtualBox, solution gratuite, multiplateforme, adaptée à l’enseignement et aux tests.
  2. Système d’exploitation : Debian, stable, léger et adapté aux serveurs.
  3. Type de machine virtuelle : architecture 64 bits supportée par le processeur de la machine hôte pour de meilleures performances.
  4. Allocation mémoire : 4 Go de RAM (possibilité de descendre à 2 Go pour un usage minimal).
  5. Nombre de processeurs : 1 cœur suffit pour les besoins de la SAÉ.
  6. Configuration réseau : mode pont pour rendre la VM accessible depuis le réseau local, permettant le test du serveur web depuis un autre appareil.
